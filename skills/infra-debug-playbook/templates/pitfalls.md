# pitfalls —— 参数/地域/入口陷阱账本（现象 → 正确做法）

> 条目格式：`## N. 一句话坑名`＋现象＋正确做法＋实测日期。新坑当场记，同坑不二踩。

## 1. `--vregion` 的位置因子命令而异，env 最稳
- 现象：`bytedcli tce instance list ... --vregion X` 与 `bytedcli tce --vregion X ...` 都可能报参数错；apm 则要求紧跟 `apm`。
- 正确做法：**优先 `export BYTEDCLI_CLOUD_SITE=i18n-tt`（海外站）用环境变量定站点**，仅在子命令明确支持时才用 `--vregion`。
- 实测：2026-07-12

## 2. 入口名 ≠ 功能名，别凭名字猜
- 现象：查数据库直觉去 `db` / `dms`，全错；正确在 `rds db query`。
- 正确做法：定位功能先读全 `bytedcli --help` 顶层域清单 + 目标域 `--help` 子命令树，再选入口；只试 1-2 个入口不许说"查不到"（self-help-first 举证义务）。
- 实测：2026-07-15（用户纠正实录，evals/cases/13）

## 3. 地域路由打错集群（mycis vs alisg/sg1）
- 现象：查询正常返回但数据为空/不对——请求路由到了错误地域。
- 正确做法：从报错/平台 URL 里的地域标识（alisg / sg1 / maliva…）确认目标地域后显式指定；空结果先怀疑地域，再怀疑数据不存在。
- 实测：2026-07-15（用户会话实录）

## 4. 指标名盲猜前缀 = 全空
- 现象：`apm grafana search restart|oom|pod` 全部 0 结果，误以为"指标不可查"。
- 正确做法：指标名从平台看板 / tagv枚举 / 服务方确认后再查；盲搜阴性 ≠ 指标不存在（阴性≠无）。
- 实测：2026-07-12
