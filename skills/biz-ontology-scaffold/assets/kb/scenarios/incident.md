# 场景卡：系统异常排查

**入口动作**
1. 定位涉事服务：`kb.py search <服务名/报错关键字>` → 找到 repo 与入口代码。
2. 查它的资源依赖：`resources/` 台账（DB/MQ/HDFS 连接、取数命令）。
3. 拿证据：按台账命令实测（pod 状态/日志/结果分布），**告警名与单计数只出假设**，定性看结果分布（成功/失败/上限）。
4. 长周期深挖（跨 session）→ 开假设账本（hypothesis-ledger 流程：UNCONFIRMED → 证据链 → CONFIRMED/REJECTED）。

**红线**：两个解释都成立先信无聊的（正常扇出 > 集群级放大）；被推翻先微复盘再给新结论。
