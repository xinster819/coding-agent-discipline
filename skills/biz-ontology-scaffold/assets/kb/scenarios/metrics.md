# 场景卡：指标异常分析

**入口动作**
1. 先对口径：该指标在 `resources/` 台账里的定义、取数命令、单位与维度（一条计数对应啥）。
2. 现拉数据（标取数时间），对齐时间轴找同时刻事件（发布/定时任务/上游波动）。
3. 证伪三问：我若错了数据长什么样？查能推翻我的字段/分布；单计数不定性。
4. 结论带置信度【高/中/低】+ 依据；口径没对齐前只出【假设】。

5. **整理/汇总类指标（"全业务 X 量"）先画"业务环节 × 数据源覆盖矩阵"**：沿业务流程盘环节（如 进审→机审→人审→审出→申诉），每环节的量在哪、"已得 / 已试未得（举证）"——完整性是一张可检验的清单，不是一句话。交付可得的最优组合，缺块显式声明；**headline 用与目标语义一致的量纲**（"送审量"的 headline 是送审对象数，机审/人审是执行量）——数据在附表、主叙事降级也算静默降级（实录：人审量纲缺席+机审任务数当 headline）。

**红线**：不拿语义未确认的计数定性；不编精确单值，给区间/条件；**交付范围 < 目标口径必须开头声明缺口**。
