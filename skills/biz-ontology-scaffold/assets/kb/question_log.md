# 问题台账 —— KB 靠真实提问生长（答得不准的，比答准的更值钱）

> 规则：**每次真实业务提问必登记一行**（验收提问强制登记）；"经验总结"只写在对话里 = 丢失,必须落到这里。
> 转化节奏：每次维护时扫一遍——业务词搜不到 → 补 `glossary.json`；同类问题出现 ≥2 次 → 升级进 `answer_contracts.json`；缺文档 → 拉进 `docs/`；缺数据通道 → `resources/` 台账补取数命令。转化完在"转化动作"列打勾留痕。

| 日期 | 提问（用户原话） | 可决策性标注 | 答准了吗（用户反馈/抽检结论） | 差在哪（缺词典/缺契约/缺文档/缺数据通道/覆盖不够） | 转化动作 |
|---|---|---|---|---|---|
| | | | | | |

> "答准了吗"列的来源有两种：用户当场反馈（少数——用户多数时候不知道答案），或**抽检**（多数）。
> 抽检 = 用 fresh context（换个不带原答案的 agent/session）对已交付答案独立走一遍证伪，结论回填本列。
