# retro：复盘 / 回顾 / 事故 postmortem

## 何时触发
- 用户说"复盘 / 周报 / 月度总结 / 阶段回顾 / 这周怎么样 / postmortem"
- 到了约定的复盘节点（每周/每迭代/里程碑）
- 一次事故或重要节点结束后

## 复盘前准备
1. **先查证再开口**：涉及数据/状态/进度/指标的结论，先用工具现查（跑统计、读 issue/PR/日志），禁止凭记忆；标注"数据截至 时间(时区)"。
2. 拉上一周期的复盘记录与当时立的行动项，用来对照"说到没做到"。

## 复盘骨架（固定结构）

### 1. 状态快照
- 本周期交付了什么 / 核心指标进展（带数据 + 时间戳 + 置信度【高/中/低】）
- 与上周期对比：变化方向用 ↑↓ / +− 显式标注

### 2. 行动项回溯
| 上期行动项 | 状态 | 说到没做到的原因 |
|---|---|---|
| … | ✅完成 / 🟡进行中 / ❌未动 | … |

### 3. 风险清单
- 已暴露风险：影响 + 当前应对
- 潜在风险：触发条件 + 预案（条件预案，而非单点预测）
- 用最坏情况压力测试关键假设
- 需要系统性的事前风险扫描时，调用 `blindspot-scan`（本节只做事后盘点，不重复其六维清单）

### 4. 关键决策回溯（反谄媚）
- 本周期做过的关键判断，事后看是否成立？
- 哪些被数据/结果证伪了？哪些当时是附和了直觉但其实站不住？
- 诚实记，不护短、不美化。

### 5. 下阶段行动项
- 可执行、不模糊：每条带负责人(我/你)、触发条件、完成判据
- 区分"我能自己干的"（包揽）与"确需你拍板的"（红灯，给求助三件套）

### （事故 postmortem 额外）
- **时间线**：何时发生、何时发现、何时恢复
- **根因**：技术根因 + 流程根因（为什么没早发现）
- **规避**：具体的防复发措施（监控/测试/流程），落到行动项

## 复盘后
- 把"被验证的学习"提议升级进 AGENTS.md / 记忆（自进化回路）。
- 机器状态（结构化指标）与人类笔记（判断/反思）分开存。
