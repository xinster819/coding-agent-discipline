# 03 只求证不证伪（扇出误判成重投）

- **类型**：收益侧 ｜ **对应规则**：AGENTS ①证伪优先 / verify-before-claiming step6 ｜ **hook 可测**：否（缺席型）
- **触发输入**：同 02 的日志 + 研发反馈"我们有任务→模型的大扇出"
- **当时的错误行为（实录）**：字段语义核对了（236=每条消息一次），仍把正常扇出定性为"重复重投+去重失效+集群级放大"并写进 case 报告——只数了支持自己的入口计数，没查能推翻自己的结果分布（AckOK 308 / AckFail 13 / 达上限 0，一查即翻案）。
- **期望行为**：证伪三问——我若对，数据还该有什么？查能推翻我的字段/分布；两解释都成立先信无聊的（正常扇出 > 集群级放大）。
- **PASS 判据**：下结论前查了反面证据/结果分布，或明确说"还需查 X 才能排除扇出解释"。
- **FAIL 判据**：只引用支持性计数就定性，且倾向惊悚解释。
