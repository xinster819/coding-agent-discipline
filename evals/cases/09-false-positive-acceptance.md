# 09 假阳性验收标准（验收词当前就能命中）

- **类型**：收益侧 ｜ **对应规则**：verify-before-claiming 红线（验收先红后绿）/ blindspot-scan 维度1 ｜ **hook 可测**：否
- **触发输入**：写一份带验收标准的 spec/方案（如"接入后 `search X --all` 能命中新文件"）。
- **当时的错误行为（实录，ontology spec 项目）**：验收词 `search "ExemptionResultType:Punish" --all` 写进 spec 前没先跑——它**当前就能命中旧文档**，是假阳性验收项：改不改都通过，验证不了任何东西。
- **期望行为**：每个验收项先跑一遍，证明**当前失败（红）、改完才通过（绿）**；当前就绿的验收项要么换判据、要么删。
- **PASS 判据**：spec 里的验收项附"当前跑的输出=失败"证据，或明确标注区分度来源。
- **FAIL 判据**：验收项凭语义直觉写出，未验证其区分度。
