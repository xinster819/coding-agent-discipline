# 正例 03：入口空间枚举后的连环推进（bytedcli rds 三步通 + 11 连查闭环）

- **场景**：被用户点拨"bytedcli 是能查的，你再试试"之后的表现（对照：此前只试 2 个入口就放弃——同一 agent，同一任务，姿势不同结果天差地别）。
- **漂亮的 move（可迁移）**：
  1. **回读能力目录**："bytedcli 的描述里明确单独列了 RDS"——答案在 help 里，重新枚举而不是重试旧入口；
  2. **连环推进不回头**：region 路由错 → 按 URL 地域标识修正 → 表结构探查 → 直捞失败样本（status=5），一口气 3+6+11 个命令把证据链走到闭环——**探索一旦启动，每步的输出决定下一步，不中途请示**；
  3. **落袋为安**：查通后先把发现写进 KB/记忆再汇报——踩通的路永久化。
- **为什么值得学**：这就是"自主探索"的样子——它不是被规则逼出来的，是**枚举入口 → 小步快跑 → 每步取证 → 沉淀回写**的节奏。规则的作用只是把"启动探索"的门槛修掉（别在第 2 步放弃）。
- **触发信号**：第一反应是"查不到/不支持"的那一刻——那正是该切入这个节奏的信号。
