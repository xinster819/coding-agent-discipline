---
description: 为跑到一半的项目生成 handoff（HANDOFF.md + tasks.md）
allowed-tools: Read, Grep, Glob, Bash(git log:*), Bash(git status:*), Bash(git diff:*)
---

你要为这个跑了一半的项目做结构化 handoff，目标是让任意新 session 能无损接手。
先调研、后产出，**只读不写**（遵守宪法·安全底线与「先读后写」）：给草稿等我确认再写盘。

【调研】
1. 扫仓库：技术栈、目录结构、关键模块、构建/测试/运行命令、外部依赖。沿用既有约定，引用 `file:line`。
2. 用 git log / status / diff 判断：已完成 / 进行中（有未提交改动）/ 待做。
3. 找出隐性信息：临时 workaround、踩过的坑、被排除的方案。

【产出两份草稿】
=== HANDOFF.md（状态快照）===
- 项目一句话目标
- 已完成（每条标对应文件/commit）
- 进行中（卡在哪 / 为什么 / 下一步打算）
- 待做（按依赖顺序）
- 已知坑 & workaround
- 关键决策记录（为什么这么选，排除了什么）
- 如何运行 / 如何测试

=== tasks.md（可勾选任务清单）===
- 把待做拆成原子任务，每条：[ ] 描述 + 完成判据 + 依赖项

【硬约束】
- 指向文件路径，不要把大段代码贴进文档（防上下文膨胀）。
- 不确定的地方标【待确认】，不要猜（宪法·先查证再开口）。
- 给完草稿停下，等我确认/补充后再写盘。
