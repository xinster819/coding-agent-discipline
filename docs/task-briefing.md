# 高预算任务简报模板（派活时复制填空）

> 适用：高价值/高预算的整理、取证、分析类任务（"全业务 X 量""彻底查清 Y"）。
> 原理：同一个 agent，任务表述不同表现天差地别——"可以 loop 100 次"写在 prompt 里没用，写成**停止条件**才有用（实证：14/100 早停 case）。

```
【目标】<结果语言，一句话。例：整理全业务送审量的权威口径表>

【完成判据】（可检验；先红后绿——现在应全是红的，做完全绿才算完成）
1. <例：覆盖矩阵里每个业务环节（进审/机审/人审/审出/申诉）标"已得"或"【尽力未得】+已试举证"，无空白项>
2. <例：headline 口径与目标语义一致（送审量=送审对象数）>
3. <例：每个数字带数据源+取数时间；高风险数字双证据链交叉>

【第一交付物】动手前先交"环节 × 数据源覆盖矩阵"草案（我确认口径再深挖，防锚定跑偏）

【预算】最多 <N> 轮循环 / <时长>。注意：预算是上限不是 KPI——
- 判据全绿 → 提前收工；
- 预算未尽但账本里还有"预算内可解"的缺口 → 不许结案；
- 预算用尽仍有缺口 → 交【尽力未得】清单（已试路径+报错+剩余未试项）。

【验收方式】<例：我抽 2 个数字要你现场复跑取数命令 / 交付前先过一遍 fresh-context 反驳>

【工具提示】先查 ~/.ai-coding-pack/infra-playbook/（任务路由/陷阱/权限）；业务口径查 KB（kb.py search / answer_contracts）；踩通新通道回写手册。
```

**Claude Code 附加动作**：把【完成判据】设为停止条件——
`/goal <目标 + 完成判据摘要>`，agent 未达判据时无法收工。
